6. Что такое git submodule ?
Git submodule - это команда, которая позволяет вам хранить git-репозиторий как подкаталог другого git-репозитория. Git submodule просто ссылается на определенный коммит в другом репозитории, не копируя его содержимое¹².

Git submodule полезен, когда вы хотите использовать внешний код или библиотеку в своем проекте и отслеживать его версионность. Это позволяет вам зафиксировать код на нужном коммите и избежать потенциальных проблем с обновлениями или несовместимостями³.

Для добавления нового подмодуля вы используете команду git submodule add с абсолютным или относительным URL проекта, который вы хотите отслеживать. Например:

```
git submodule add https://github.com/chaconinc/DbConnector
```

Эта команда добавит репозиторий DbConnector как подкаталог в вашем проекте. По умолчанию, подмодуль будет иметь то же имя, что и репозиторий, но вы можете указать другой путь в конце команды, если хотите¹.

При добавлении подмодуля будет создан новый файл .gitmodules, который содержит информацию о соответствии между URL проекта и локальным каталогом, в который он загружен. Если у вас несколько подмодулей, то в этом файле будет несколько записей. Этот файл является версионируемым с вашими другими файлами¹.

Для обновления подмодуля вы используете команду git submodule update с опциональными параметрами. Например:

```
git submodule update --remote --merge
```

Эта команда обновит все подмодули до последних доступных коммитов из удаленных репозиториев и объединит их с вашими локальными копиями¹.

Для удаления подмодуля вы должны выполнить несколько шагов:

- Удалить соответствующую запись из файла .gitmodules.
- Удалить соответствующую запись из файла .git/config.
- Выполнить git rm --cached <path_to_submodule> (без косой черты в конце).
- Зафиксировать изменения.
- Удалить физический каталог подмодуля из вашего проекта.

Вы можете использовать разные опции с git submodule, например:

- --init: инициализировать подмодуль при добавлении или обновлении.
- --recursive: рекурсивно обновить все вложенные подмодули.
- --force: принудительно перезаписать локальные изменения при обновлении.

Подробнее об этих и других опциях вы можете прочитать в документации по git submodule.

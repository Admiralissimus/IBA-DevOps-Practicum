5. Какая разница между git pull и git fetch ?
Git pull и git fetch - это две команды, которые позволяют загружать данные из удаленного репозитория в локальный. Однако они делают это по-разному¹².

Git fetch просто скачивает новые объекты и ссылки из удаленного репозитория, но не интегрирует их в вашу рабочую копию. Git fetch полезен для получения свежего взгляда на все, что произошло в удаленном репозитории. Это безопасная операция, которая никогда не изменяет ваши локальные ветки³.

Git pull, в отличие от git fetch, обновляет вашу текущую ветку с последними изменениями из удаленного репозитория. Это означает, что git pull не только скачивает новые данные, но и непосредственно интегрирует их в ваши рабочие файлы. Это имеет несколько последствий:

- Поскольку git pull пытается объединить удаленные изменения с вашими локальными, может возникнуть так называемый \"конфликт слияния\". Подробнее о том, как разрешать такие конфликты, вы можете прочитать в нашем подробном учебнике о том, как справляться с конфликтами слияния.
- Как и для многих других действий, настоятельно рекомендуется начинать git pull только с чистой рабочей копии. Это означает, что у вас не должно быть никаких незафиксированных локальных изменений перед тем, как вы делаете pull. Используйте функцию Git Stash, чтобы временно сохранить свои локальные изменения.

Пример использования git fetch:

```
git fetch origin
```

Эта команда скачает все объекты и ссылки из удаленного репозитория origin в ваш локальный репозиторий. Вы можете посмотреть список удаленных веток с помощью git branch -r. Если вы хотите посмотреть историю какой-либо удаленной ветки, вы можете использовать git log origin/<branch>. Если вы хотите переключиться на какую-либо удаленную ветку, вы можете использовать git checkout --track origin/<branch> (это создаст локальную ветку, которая будет отслеживать удаленную)³.

Пример использования git pull:

```
git pull origin main
```

Эта команда скачает ветку main из удаленного репозитория origin и объединит ее с вашей текущей веткой (если она соответствует). Это обновит вашу локальную копию main до последней версии¹.

Выбор между git pull и git fetch зависит от того, какую цель вы преследуете. Если вы хотите просто получить информацию о том, что происходит в удаленном репозитории, без изменения своих рабочих файлов, то лучше использовать git fetch. Если вы хотите быстро обновить свою текущую ветку до последней версии из удаленного репозитория, то лучше использовать git pull².
